# Team 5 Final Project - Real-time Cryptocurrency Monitoring System (English Version)

## Project Overview
An automated web server construction project utilizing Docker Compose and external APIs.
Provides real-time cryptocurrency price information and exchange rate data through periodic updates on a web page.

## Planning

### Reasons for Topic Selection
- **Real-time Nature**: Cryptocurrencies operate 24/7 with constant price fluctuations, making real-time monitoring crucial
- **Practicality**: Provides KRW conversion functionality for Korean users
- **Technology Application**: Comprehensive utilization of course content including Docker, API, Cron
- **Differentiation**: Accurate KRW calculation with real-time exchange rates rather than simple display

### Objectives
- Real-time monitoring of top 12 cryptocurrencies
- Accurate KRW calculation with real-time exchange rates
- Automated updates every 30 seconds for latest information
- Intuitive and modern UI/UX
- Multi-backup system for stable service operation

## Implementation Description

### System Architecture
The system consists of multiple containers orchestrated through Docker Compose:
- **crypto-monitor Container**: Core data processing and HTML generation
- **nginx Container**: Web server for static file serving
- **CoinGecko API**: Cryptocurrency data source
- **ExchangeRate-API**: Real-time USD/KRW exchange rates
- **Backup System**: Local file-based data recovery mechanism

### Core Features

#### 1. Backup System for API Instability Resolution
**Problem**: Data loss and service interruption due to intermittent API failures
**Solution**:
- **Data Backup Storage**: Automatic local storage of successful API responses
- **Auto Recovery**: Immediate switch to backup data upon API failure
- **Status Visualization**: Clear indication of real-time/backup data status to users
- **Service Continuity**: Always provide data regardless of API failures

#### 2. Real-time Data Collection and Recovery
- **Cryptocurrency Data**: Price, ranking, 24-hour change rate, logos
- **Exchange Rate Data**: Real-time USD/KRW exchange rates
- **Data Validation**: JSON validity checking using jq
- **Auto Backup**: Backup file updates with each successful operation

#### 3. Automation System (API Stability Considered)
- **Cron Design**: 30-second intervals implemented with 2 jobs (reduced API load)
- **Execution Cycle**: Runs at 0 and 30 seconds of each minute
- **Immediate Execution**: First data collection upon container startup
- **Load Distribution**: Reduced API call frequency for enhanced stability

#### 4. Data Processing and Formatting
- **JSON Parsing**: Accurate parsing using jq library
- **Exchange Rate Calculation**: Precise decimal calculations using bc command
- **Number Formatting**: Thousand separators, optimized decimal places
- **Error Handling**: Default values upon calculation failures

#### 5. Modern Web Interface
- **Responsive Design**: CSS Grid layout
- **Glassmorphism**: Transparency and blur effects
- **Status Display**: Color coding based on price fluctuation (rise/fall/neutral)
- **Auto Refresh**: Automatic page updates every 30 seconds
- **Backup Alerts**: Clear status indication when using previous data

## Technology Stack and Selection Rationale

### Containerization
- **Docker**: Environment independence guarantee
- **Docker Compose**: Multi-container orchestration
- **Volume Sharing**: Data sharing between containers

### Web Server
- **nginx**: Optimized for static file serving, high performance
- **Port Mapping**: External access through port 80

### Scheduling and Scripting
- **cron**: Linux standard scheduler
- **bash**: System standard script, no additional packages required
- **curl**: Standard tool for HTTP requests
- **jq**: Professional tool for JSON parsing

### API Selection
- **CoinGecko API**: Free, reliable, complete data
- **ExchangeRate-API**: Stable exchange rate service (main)
  - Initially planned to use Korea Export-Import Bank exchange rate API
  - Changed due to instability and response delay issues
  - ExchangeRate-API provides more stable and faster responses
- **Backup System**: Local file-based data recovery

## Creative Technical Implementations

### 1. 30-Second Cron Interval Implementation
```bash
# Optimized for 30-second intervals to reduce API load
echo "*/1 * * * * root /usr/local/bin/update.sh" > /etc/cron.d/crypto-cron
echo "*/1 * * * * root sleep 30 && /usr/local/bin/update.sh" >> /etc/cron.d/crypto-cron
```

### 2. Automated Backup Data Management System
```bash
# Save backup on success
cp /tmp/crypto_data.json "$LAST_SUCCESS_FILE"
echo "$usd_rate|$exchange_source|$(date)" > "$LAST_EXCHANGE_FILE"

# Restore backup on failure
if [ -f "$LAST_SUCCESS_FILE" ]; then
    cp "$LAST_SUCCESS_FILE" /tmp/crypto_data.json
fi
```

### 3. Accurate JSON Parsing with jq
```bash
# Enhanced stability using jq instead of regular expressions
name=$(jq -r ".[$i].name" /tmp/crypto_data.json)
price=$(jq -r ".[$i].current_price" /tmp/crypto_data.json)
```

### 4. Dynamic Formatting System
```bash
# Different decimal handling based on price range
if (( $(echo "$price >= 1" | bc -l) )); then
    formatted_price=$(printf "%.2f" "$price")  # $1.00
else
    formatted_price=$(printf "%.6f" "$price")  # $0.000001
fi
```

### 5. Log Management through stderr/stdout Separation
```bash
# Logs to stderr, return values to stdout
log "üí± Collecting exchange rate information..." >&2
echo "$usd_rate|$exchange_source"  # Pure return value only
```

## Technical Challenges and Solutions

### Challenge 1: API Instability and Service Interruption Risk
**Problem**: Data loss due to intermittent CoinGecko API and exchange rate API failures
**Solution**:
- **Backup Data System Construction**: Automatic local storage of successful API responses
- **Auto Recovery Mechanism**: Immediate switch to backup data upon API failure
- **Status Visualization**: Clear indication of real-time/backup data status to users
- **Service Continuity Guarantee**: Always provide data regardless of API failures

### Challenge 2: Cross-platform Compatibility for Time Handling
**Problem**: GNU date's `-d` option not supported on Windows, causing time calculation errors
**Solution**:
- **Docker Container Environment**: Platform independence through containerization
- **Platform-compatible Logic**: Modified time calculation logic for cross-platform compatibility
- **Explicit Timezone Setting**: Specified timezone settings for consistency
- **Comprehensive Testing**: Problem discovery and resolution through Windows testing by team members

### Challenge 3: Exchange Rate API Selection and Stability Issues
**Problem**: Instability and response delays of Korea Export-Import Bank exchange rate API
**Solution**:
- **API Testing Results**: Discovered response speed degradation and intermittent failures
- **API Migration**: Significantly improved stability and response speed with ExchangeRate-API
- **24/7 Stable Service**: Reliable exchange rate information through international API
- **Dual Safety Mechanism**: Combined with backup system for enhanced reliability
